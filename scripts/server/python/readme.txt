Небольшое пояснение к Digital_sign.py. Для создания открытого ключа используется класс PublicKey. После создания открытого ключа на его основе нужно создать закрытый при помощи класса PrivateKey. Класс Message не предполагает создания экземпляра, он необходим для преобразования строки. Шифрование строки происходит при помощи метода encode, на вход которому подаётся текст и открытый ключ. Генерация цифровой подписи осуществляется тем же методом, но на вход подаётся не открытый, а закрытый ключ. Дешифрование и расшифровка подписи осуществляются методом decode, на вход подаётся зашифрованный тест (список чисел) и закрытый ключ (при дешифровании зашифрованного сообщения), либо открытый ключ (при расшифровке подписи).
